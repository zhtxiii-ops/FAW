# Fractal Agent Workflow (FAW)

## 项目简介

FAW（Fractal Agent Workflow）是一个基于递归 Map-Reduce 架构的多智能体工作流系统。它能够将复杂的自然语言任务，通过大语言模型（LLM）自上而下地拆分为细粒度的子任务树，并在执行过程中通过严格的数据契约（Schema）、沙箱环境以及实时监控，确保各个子任务节点的稳定执行和结果交付。

最新架构引入了**单次调度（One-Shot Planning）**与**流式前端（Streaming Tree UI）**，有效减少了节点之间的冗余通信，并提升了任务执行状态的可观测性。

## 核心特性

- **单次任务规划 (One-Shot Execution Planning)**：取消了传统的多阶段规划步骤。由核心调控器单次 LLM 推理直接生成强约束的 `ExecutionPlannerResult` 数据模型，评估任务难度（SIMPLE / COMPLEX / UNKNOWN）并规划执行路径。
- **嵌套递归拆解 (Recursive Map-Reduce)**：针对复杂任务（COMPLEX），系统自动将其拆解为相互依赖的有向无环图（DAG），分配并发执行序列，并在子任务完成后进行归约汇总。系统内置最大深度限制（Max Depth），防止陷入无限递归。
- **统一验收器 (Unified Validator)**：采用"代码优先、LLM 兜底"策略。规划器仅用自然语言描述验证需求，专职代码生成模型自动将其翻译为 Python 验证代码执行；无法代码化的主观性需求则兜底调用 LLM 审核。配合 Schema 格式校验，形成高可靠的多层防御体系。
- **异步技能合成 (Asynchronous Skill Synthesis)**：除内置工具（如计算器、网络搜索、文本处理）外，系统支持按需扩展。当所需工具不存在时，独立工作的开发智能体会自动介入，专门负责编写、测试该工具，验证通过后系统将热加载代码并持久化保存，从而实现真正的智能体自我进化赋能。
- **流式可视化控制台 (Streaming Web Console)**：提供基于 FastAPI 和 Uvicorn 驱动的异步 Web 前端。运行日志通过异步队列与 NDJSON 事件流实时推送到前端，可视化展示任务递归树和实时进度。

## 工作流程

FAW 的核心处理流程遵循自顶向下的“拆解-执行-合并”范式，具体步骤如下：

1. **输入解析 (Input Analysis)**：接收用户的自然语言指令，提取核心诉求与关键参数。
2. **单次规划 (One-Shot Planning)**：核心大语言模型（LLM）评估任务复杂度并返回具有强数据类型（Schema）约束的规划方案。
   - **SIMPLE**：若为简单任务，直接构建单节点执行计划。
   - **COMPLEX**：若为复杂任务，将其裂变为相互依赖的子任务有向无环图（DAG），并明确并发执行序列。
   - **UNKNOWN**：若为意图不明探索性任务，则生成信息收集型前置节点，待探测完毕后重新规划。
3. **并发调度 (Concurrent Execution)**：依据 DAG 的依赖关系，异步调度各子任务节点。当规划器请求了一个工具库中不存在的临时工具名称时，自动拉起开发智能体合成对应 Python 技能脚本，验证通过后热加载执行。
4. **统一验收 (Unified Validation)**：对于每个节点的产出，系统均会实施"代码优先、LLM 兜底"的统一验收。规划器用自然语言描述的验证需求被自动翻译为 Python 代码执行；无法代码化的主观性需求则交由 LLM 审核。
5. **归结汇总 (Reduction)**：当所有子节点均成功完成推演后，系统根据特定的归约逻辑（Reduction Logic）对局部数据进行缝合汇总，合成最终交付结果。
6. **状态同步 (State Streaming)**：在系统的整个生命周期内，所有节点的日志和状态演变都会实时进入异步机制，经由事件流推送至 Web 控制台前端，动态绘制形成可视化推演拓扑。

## 优缺点评价

### 🌟 优势 (Pros)
- **极强的自主拆解与编排能力**：通过先进的单次 LLM 规划（One-Shot execution planning），系统自动将宏大、模糊的目标裂变为高并发的子任务树，极大降低了用户指令编排的心智负担。
- **高鲁棒性的防御机制**：独创的三重验收体系（Schema 格式强约束 + 动态生成 Python 代码验证 + LLM 感性语义兜底）大幅减少了底层大模型产生幻觉的概率，保障了最终合并交付物的严谨性。
- **自我进化的技能扩容**：系统不仅提供内置的原子工具箱，当面临需要全新未注册工具的场景时，能够无缝拉起开发智能体，现场生成并持久化保存 Python 技能代码，实现了系统“行动能力的自我增殖”。
- **白盒化的异步流式前端**：借助 Uvicorn 和 Server-Sent Events (SSE)，使得黑盒的大模型深层递归推理过程变得完全白盒化，开发者能够在图形化 Web 控制台上实施全链路实时追踪。

### ⚠️ 局限性与挑战 (Cons)
- **过度拆解的“大公司病”**：系统过于依赖单一中心模型（如 DeepSeek-v3.2 / GPT-4）的单次拆解规划能力。针对一些复合概念，高智商模型容易本能地产生过度发散的分解欲，导致任务图谱层层“外包”，如果触碰最大深度限制（Max Depth）会引发强制降级熔断。
- **算力和 Token 消耗极速膨胀**：由于 Map-Reduce 架构会裂变出庞大的并发兵团，且任务的每一层都伴生着强制的审查（Review）、重试和数据适配（Comprehension），一次深层复杂的调研可能会在几分钟内瞬发数十次大模型高并发请求，对 API 的额度和限流防御策略提出了极高的要求。
- **严苛的契约导致的死锁风险**：强 Schema 和代码级验证的“原教旨主义”虽然提高了数据的准确上限，但在面对非结构化搜索或高度模糊的发散型任务时，经常会因为非常严苛的代码验证条件（如缺乏宽松的正则匹配）导致其实早已正确收集到的情报被反复拦截、重试乃至阵亡。

## 项目结构

```text
FAW/
├── main.py                # CLI 入口
├── web_app.py             # Web 控制台服务（FastAPI）
├── start.sh               # 前端与服务启动脚本
├── base_agent.py          # 核心流程控制（单次调度、Map-Reduce 拆解图构建）
├── models.py              # 系统核心数据模型（Pydantic Schema）
├── reviewer.py            # 结果审核与校验逻辑
├── skill_manager.py       # 技能加载与调度中心

```

## 快速上手

### 1. 环境准备

推荐使用 Python 3.12 及以上版本。克隆项目后，安装必要的依赖：

```bash
pip install -r requirements.txt
```

### 2. 环境变量配置

系统默认对接兼容 OpenAI 接口规范的 LLM 模型。请在项目根目录创建 `.env` 文件，或直接在终端中配置以下环境变量：

```bash
# 模型名称
export FAW_LLM_MODEL="deepseek-chat"
# 你的 API Key
export OPENAI_API_KEY="sk-xxxxxxxxxxxxxxxxxxxxxxxx"
# 模型 Base URL
export OPENAI_BASE_URL="https://api.deepseek.com"
```

### 3. 运行方式

系统提供 Web 控制台和命令行界面（CLI）两种交互方式。

#### 选项 A：启动 Web 控制台（推荐）

直接执行提供的启动脚本：

```bash
./start.sh
```

服务启动后，在浏览器访问 [http://127.0.0.1:8000](http://127.0.0.1:8000) 即可通过图形化界面输入任务并实时查看执行推理拓扑图。

#### 选项 B：使用命令行 (CLI)

用于快速执行或查看底层调试信息：

```bash
python main.py "分步计算 3+4，然后再计算 10*5，最后把他们加起来。" --debug-tasks --debug-llm
```

## 单元测试

核心代码修改后，请使用 pytest 运行系统级测试用例，以验证沙盒边界配置（如异常回溯、深度限制截断等）是否正常工作：

```bash
python -m pytest tests/ -v
```
